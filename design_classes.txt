Strategy - данный класс проектирования можно использовать при реализации классов Bonus или CombinationСhecker, чтобы переключать логику обработки разных бонусов/комбинаций.
FactoryMethod - данный класс проектирования можно использовать для генерации различных видов бонусов, начального состояния поля(GameBoard) или новых ячеек поля(Filed).
State - данный класс проектирования можно использовать для управление фазами игры(ожидание хода, обработка, падение, завершение), то есть при реализации класса GameLogic(игровая логика).
